Testing and debugging

    Errors are caused when something goes wrong in a program. They are usually caused by one of the following:

  • System error ― problem with the system or external devices.
  • Programmer error ― the program contains incorrect syntax, as simple as a typo.
  • User error ― the user has entered data incorrectly.

    • Exceptions are errors that produces a return value that can then be used by the program to deal with the error. For
    example, trying to call a method that is nonexistent.
    Stack Traces: An exception will also produce a stack trace. This is a sequence of functions or method calls that lead
    to the point where the error occurred.

    Debugging Tools Most modern browsers (Firefox, Edge, Chrome, Safari) also have a debugging tool that allows you to set breakpoints in your code that will pause it at certain points. One of the most useful commands is the debugger keyword. This will create a breakpoint in your code that will pause the execution of the code and allow you to see where the program is currently up to. You can also hover over any variables to see what value they hold at that point.
